# Healthcare-saas-analytics-project

## Project Overview:

This project simulates a real-world Healthcare SaaS analytics environment. The goal of the project is to analyze product usage, customer behavior, billing, and support data in order to answer important business questions and generate insights that can guide product, customer success, and revenue strategies.

The project demonstrates how SQL and data visualization tools can be used to transform raw healthcare platform data into meaningful business insights.

## Objectives:

The main objectives of this project are to:

Analyze healthcare customer accounts and user behavior
Understand product feature adoption and engagement
Identify revenue patterns from billing data
Explore customer support trends
Generate business insights that could inform decision-making
Build a Power BI dashboard to visualize key metrics

## Tools Used:

SQL – Data querying and analysis
Power BI – Data visualization and dashboard creation
GitHub – Project documentation and version control

## Key Business Questions:

- What is the total revenue generated, and how does it break down by hospital size?
- What percentage of accounts are being renewed, pending, or churned?
- Which regions contribute the most revenue, and where are growth opportunities?
- How does renewal performance fluctuate over time, and what patterns suggest risk or opportunity?
- What is the financial impact of churned accounts compared to renewed ones
- Are users actively engaging with the platform after onboarding?
- Which regions and departments show the strongest adoption, and where is improvement needed?
- Is workforce adoption balanced across roles?
- How does user activity evolve over time?

## SQL Analysis

SQL queries were used to explore and analyze the data across multiple tables including:

Accounts
Users
Product usage events
Billing records
Support tickets

The queries include aggregations, joins, filtering, and grouping to uncover meaningful trends in the data.
All SQL scripts used in this project are located in the SQL folder.

## Dashboard

A Power BI dashboard was created to visualize key metrics such as:

User activity and engagement
Revenue by account
Product feature usage
Customer support trends

The dashboard helps translate raw data into a visual format that supports data-driven decision making.

## Key Insights:

- Healthy Adoption: 2K total users with a 75% Active User Rate shows strong engagement and effective onboarding.
- Balanced Workforce: Adoption is spread across Admins, Nurses, and Doctors, ensuring the platform is relevant to multiple healthcare roles.
- Departmental Balance: Oncology, Radiology, ER, ICU, and Cardiology all contribute nearly evenly to active usage, proving the system’s broad applicability.
- Regional Differences: Some regions (South, West) show stronger activity than others (Northeast, Midwest), highlighting where engagement strategies should be targeted.
- Trends Over Time: Monthly fluctuations in Active User Rate reveal seasonal cycles or product adoption challenges, useful for anticipating churn or planning interventions.

Further detailed insights are documented in the Analysis folder.
